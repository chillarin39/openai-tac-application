# TAC 利用ログ

OpenAI Trusted Access for Cyber (TAC) の利用記録。月次サマリで利用回数 / 用途カテゴリ別件数 / ポリシー逸脱の有無を記録し、将来の GPT-5.5 Cyber 上位申請時の track record とします。

**運用ポリシー**: [docs/ai-usage-policy.md](./ai-usage-policy.md)
**有効化日**: 2026-05-30 (chatgpt.com/cyber での KYC 通過)

---

## 月次サマリ運用ルール

- 毎月末に当月分のサマリを「月次サマリ」セクションへ追記する (新しい月を上に積む)
- 利用回数の粒度は「セッション数」(同じ話題の往復は 1 セッションと数える)
- 用途カテゴリは [ai-usage-policy.md §9](./ai-usage-policy.md) の Phase 区分と整合させる:
  - **Phase 1: 対話型** (記事執筆補助 / CVE 調査 / 検知ルール添削)
  - **Phase 2: 読み取り中心** (Git PR レビュー / IaC レビュー)
  - **Phase 3: 限定 HTTP 接続** (隔離セグメント内ターゲットへの偵察補助)
  - **Phase 4: 隔離環境演習** (完全隔離セグメント内の承認付き演習)
- ポリシー逸脱は原則「該当なし」と記録し、何か起きた場合は詳細を記述したうえで [ai-usage-policy.md §10](./ai-usage-policy.md) の対応フローを実行する
- Codex Security を自リポジトリ監査に使った場合は、リポジトリ名 + 主な findings カテゴリを 1 行で記録する (詳細は別途)
